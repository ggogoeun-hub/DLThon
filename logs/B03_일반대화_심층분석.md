# B03 일반대화 분류 심층 분석

> 모델이 500건 test 중 32건만 일반대화로 분류하는 원인 분석

---

## 1. 일반대화 확률 분포 — 모델의 인식 수준

| 일반확률 기준 | 해당 건수 | 의미 |
|---|---|---|
| > 0.5 (일반으로 분류) | **31건** | 모델이 확신하는 일반대화 |
| 0.3 ~ 0.5 | 2건 | 경계선 — 살짝만 밀면 일반으로 넘어감 |
| 0.1 ~ 0.3 | 1건 | 모델이 약간 고민함 |
| 0.05 ~ 0.1 | 3건 | 거의 위협으로 확신 |
| < 0.05 | **462건 (92.4%)** | 모델이 일반대화 가능성을 완전히 무시 |

**핵심**: 모델은 500건 중 462건(92%)에 대해 일반대화 확률을 5% 미만으로 판단. 경계 사례(0.05~0.5)는 고작 6건. 즉, **모델의 결정 경계가 극단적으로 위협 쪽으로 치우쳐 있음**.

---

이 6건이 가장 중요합니다. 임계값 조정으로 일반으로 뒤집을 수 있는 건들입니다.

## 3. 모델이 어려워하는 3가지 패턴

### 패턴 A: 위협 키워드가 비유적으로 사용된 경우
```
"일 폭탄으로 누구 죽일 수 있어" → 협박으로 분류
실제: 업무 스트레스 표현 (비유)
```
모델은 "죽일", "폭탄" 같은 키워드에 과민 반응. 문맥(업무 스트레스)을 파악하지 못함.

**학습 데이터에 없는 것**: 위협 키워드가 비유적으로 사용된 일반 대화

### 패턴 B: 신고/긴급 상황 대화
```
"화재 발생... 다친 사람 있나요? 119 소방대를 보내겠습니다" → 협박으로 분류
"시신을 발견했습니다. 인천 영흥도 해상" → 협박으로 분류
"옥상에서 사람이 떨어졌어요" → 기타괴롭힘으로 분류
```
신고/긴급 대화는 위협 키워드(화재, 시신, 떨어졌어요)가 포함되지만 위협이 아님. 이 유형은 합성 일반 데이터에 전혀 포함되지 않음.

**학습 데이터에 없는 것**: 신고/긴급/사건 대화

### 패턴 C: 직장/관계 맥락의 정상 대화
```
"좋은 아침입니다 이동진 대표님" → 기타괴롭힘으로 분류 (존댓말+직장 = 괴롭힘 숏컷)
"짐은 다 뺐니 택배로 부쳤어요" → 직장괴롭힘으로 분류
"지갑 잃어버렸다 현금 15만원" → 갈취로 분류 (돈 관련 = 갈취 숏컷)
```
직장 맥락의 존댓말 대화, 돈이 언급되는 일상 대화를 위협으로 분류. Hard Negative 전략이 아직 부족함.

**학습 데이터에 부족한 것**: 직장 인사/돈 관련 일상 대화 (더 많은 Hard Negative 필요)

---

## 4. 위협 키워드 없는데 위협으로 분류된 건: 239/468건 (51%)

위협으로 분류된 468건 중 **절반 이상(239건)**이 명시적 위협 키워드 없이 위협으로 분류됨. 이는 모델이 키워드가 아닌 **문체적 특성(마침표, 존댓말, 길이)**으로 분류하고 있다는 추가 증거.

---

## 5. 클래스별 확신도 — 모든 클래스가 과신 상태

| 클래스 | 건수 | 평균 확신도 | 최소 | 불확실(<0.5) |
|--------|------|-----------|------|------------|
| 갈취 | 107 | 0.917 | 0.519 | 0건 |
| 기타괴롭힘 | 142 | 0.906 | 0.430 | 2건 |
| 일반 | 32 | 0.903 | 0.488 | 1건 |
| 직장괴롭힘 | 113 | 0.925 | 0.337 | 2건 |
| 협박 | 106 | 0.918 | 0.388 | 1건 |

모든 클래스 평균 확신도 90%+ → Label Smoothing 0.1에도 불구하고 여전히 과신. 불확실 예측(<0.5)은 총 6건뿐.

## 6. 근본 원인 요약

```
┌─────────────────────────────────────────────────┐
│          모델이 일반대화를 못 찾는 이유           │
├─────────────────────────────────────────────────┤
│                                                 │
│  1. 합성 일반 ≠ 실제 일반 (문체 숏컷)            │
│     마침표/존댓말 분포 차이가 여전히 큼           │
│     보정 후에도 마침표 3.0 vs 위협 6.2           │
│                                                 │
│  2. 학습 데이터에 없는 유형                      │
│     - 신고/긴급 대화 (119, 경찰)                │
│     - 비유적 위협 표현 ("일 폭탄", "죽겠다")    │
│     - 돈/직장 맥락 일상 대화 (Hard Neg 부족)    │
│                                                 │
│  3. 증강 데이터가 결정 경계를 왜곡               │
│     위협 클래스 ×3.4 증강 → 위협 쪽 과적합      │
│     462/500건을 일반확률 <5%로 판단              │
│                                                 │
│  4. 과신 (Label Smoothing 효과 부족)            │
│     평균 확신도 90%+ → 경계 사례가 6건뿐        │
│                                                 │
└─────────────────────────────────────────────────┘
```

## 7. 개선 방향 (우선순위)

| # | 방향 | 구체적 행동 | 기대 효과 |
|---|------|-----------|----------|
| 1 | **합성 일반에 신고/긴급 대화 추가** | 119 신고, 사건 신고, 응급 상황 대화 50~100건 | 패턴 B 해결 |
| 2 | **비유적 위협 표현 Hard Negative** | "일 때문에 죽겠다", "시험 폭탄" 등 50건 | 패턴 A 해결 |
| 3 | **돈/직장 Hard Negative 추가** | "지갑 잃어버림", "월급 얼마", "대표님 인사" 50건 | 패턴 C 해결 |
| 4 | **Label Smoothing 0.1 → 0.2** | 더 강한 과신 억제 | 경계 사례 증가 |
| 5 | **Confidence Threshold 도입** | max_prob < 0.7이면 2순위 클래스 고려 | 6건 경계→일반 전환 |
| 6 | **증강 비율 축소 (×3.4 → ×1.5)** | 위협 과적합 완화 | 결정 경계 균형 회복 |

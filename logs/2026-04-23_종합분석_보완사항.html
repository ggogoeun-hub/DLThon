2. 긴급 보완 사항 (즉시 조치)

보완 1: 합성 데이터 이모티콘 숏컷 문제

긴급 strategy_v1.md에서 경고한 내용이 현재 반영되지 않음

항목원본 Train우리 합성 데이터위험
이모티콘 (ㅋㅋ, ㅎㅎ, ㅠㅠ)0.0건 (없음!)대부분 포함모델이 "이모티콘 있음 = 일반 대화" 숏컷 학습
말줄임 (...)0.0건일부 포함동일 위험

원인: 원본 train 데이터의 emoticon_len = 0.0인데, 합성 데이터 생성 시 "자연스러운 대화"를 위해 ㅋㅋ, ㅎㅎ을 넣었음. 이로 인해 모델은 이모티콘 유무만으로 일반/위협을 구분하는 숏컷을 학습.

해결: 합성 데이터에서 모든 이모티콘/초성 패턴 제거 후 재학습. 또는 이모티콘 유지/제거 Ablation 실험.

import re
def remove_emoticons(text):
    # ㅋ, ㅎ, ㅠ, ㅜ 반복 패턴 제거
    text = re.sub(r'[ㅋㅎㅠㅜ]{2,}', '', text)
    # ... 말줄임 제거
    text = re.sub(r'\.{2,}', '', text)
    return re.sub(r'\s+', ' ', text).strip()

보완 2: 제출 파일 레이블 형식 확인

긴급 숫자 레이블 제출 시 일반 대화(4) 누락 버그

submit_num_order.csv에 NaN이 23건 존재:

해결: Kaggle 제출 시 어떤 레이블 형식을 요구하는지 재확인. NaN이 있는 파일로 제출하면 F1 점수에 큰 손실.

보완 3: Val/Test 괴리 (과적합)

긴급 모든 실험에서 Val은 높지만 Test 일반대화 탐지가 저조

지표B01 (세션1)B02 (세션2)해석
Val 일반대화 F10.995~0.99+두 세션 모두 "완벽"
Test 일반대화 예측34건 / 100건23건 / 100건실제 성능은 매우 낮음

원인: Validation에 사용하는 일반 대화도 합성 데이터에서 분할한 것. 합성 데이터끼리는 잘 구분하지만, 실제 일반 대화는 분포가 다름. Validation 자체가 실제 성능을 반영하지 못함.

해결: 합성 데이터의 스타일을 실제 대화에 맞추거나, pseudo-labeling으로 Test 예측 → 고신뢰 샘플을 훈련에 추가하는 self-training 접근.

6. 권장 액션 플랜 (우선순위 순)

P0 합성 데이터 이모티콘 제거 후 재학습

가장 쉽고 빠른 개선. 정규식 한 줄로 ㅋㅋ/ㅎㅎ/ㅠㅠ 제거 → 재학습. 예상 소요: 40분

P0 합성 데이터 도메인 비율 조정 (직장 30% / 학교 20%)

Test에서 직장 내 괴롭힘/기타 괴롭힘과 혼동되는 일반 직장/학교 대화 비율 확대. 현재 10% → 30%/20%로.

P1 max_len=256 학습 완료 (v3 결과 활용)

v3(46건)이 현재 최고. 체크포인트가 있으므로 해당 모델로 제출 파일 확인.

P1 합성 데이터 3,000건으로 확대

strategy_v1.md 목표. 위협 클래스 증강(3,000건/클래스)과 균형 맞추기.

P1 위협 클래스 증강 (패러프레이즈 + 역번역)

원본 ~950건 → 3,000건 목표. 문장 셔플은 에스칼레이션 패턴 파괴로 제외.

P2 모델 비교 (KcBERT / KLUE-RoBERTa)

KcELECTRA 외 다른 모델의 일반 대화 탐지 성능 비교. 앙상블 후보 확보.

P2 Focal Loss / Label Smoothing 실험

협박↔기타괴롭힘 유사도 0.87로 가장 혼동되는 쌍. Focal Loss로 완화 시도.

1. 데이터는 왜 이렇게 만들었나

출발점: B02의 참사

15,000건으로 학습 → Val F1 99.02% → Kaggle Test F1 0.03

모델이 대화 "내용"이 아니라 합성 데이터의 "문체"를 학습했다.
숏컷합성 일반위협모델이 배운 규칙

전략 전환: Top-Down → Bottom-Up

v1 (실패) "위협을 분석해서 반대를 만들자"

→ 위협에 없는 특성(반말, 구두점 없음)으로 합성 → 문체 숏컷 유발

v2 (B04) "실제 일반 대화가 뭔지 파악하고 재현하자"

→ test 일반 대화 104건 분석(correct.md) → 실제 특성을 33도메인으로 재현

핵심 발견 (correct.md): 실제 일반 대화에는 비속어("야 새끼 빨리 나와"), 관용 위협("야 죽을래"), 119 신고 전화가 포함됨. v1에서 이 모든 것을 금지했기 때문에 모델이 이런 test 데이터를 위협으로 오분류.

33개 도메인 설계 — 왜 5개에서 33개로?

B03 심층 분석에서 모델이 실패하는 3가지 패턴을 발견. 각 패턴에 대응하는 도메인을 설계.

실패 패턴 A 비유적 위협 표현 → "죽겠다"(피곤) 를 협박으로 분류

→ 대응 도메인 P09(술자리), P20(야근 불만), P22(취업 스트레스) — 과장된 하소연 포함

실패 패턴 B 신고/긴급 대화 → "화재 발생" 신고를 협박으로 분류

→ 대응 도메인 P29(119 화재/사고), P30(112 분실/사건) — v1에 전혀 없던 유형

실패 패턴 C 돈/직장 맥락 일상 → "지갑 분실 15만원"을 갈취로 분류

→ 대응 도메인 P12(친구 간 돈 얘기 HN), P15(커플 다툼 HN), P17(형제 대화 HN)

Group도메인 수건수역할
A. 친구 수다13개~1,183음식, 게임, 쇼핑, 돈(HN) 등 일상 커버
B. 커플/가족5개~455부모-자녀, 형제(HN), 커플 다툼(HN)
C. 직장인5개~455꼰대 뒷담, 야근 불만, 알바 — 비유적 위협 포함
D. 온라인/일상5개~455카톡, 동창, 택배, 건강, 이사
E. 신고/상담5개~455119, 112, 감정 상담 — 패턴 B 대응

2. 모델은 왜 RoBERTa인가

B01~B03의 근본 한계

B03 심층 분석: 위협 키워드 없는데 위협으로 분류된 건 = 239/468건 (51%)
→ 모델이 단어가 아닌 문체 패턴으로 분류. 맥락(누가→누구에게, 의도, 결말)을 파악 못함.

같은 단어, 다른 맥락 — KcELECTRA가 구분 못하는 사례:

텍스트실제KcELECTRA 판단
"일 때문에 죽겠다"업무 스트레스 (일반)협박
"지갑 잃어버렸다 15만원"분실 (일반)갈취
"좋은 아침입니다 대표님"직장 인사 (일반)괴롭힘
"화재 발생 119 소방대"신고 (일반)협박

KcELECTRA vs KLUE-RoBERTa — 구조적 차이

KcELECTRA (B01~B03)KLUE-RoBERTa (B04)
사전학습 방식RTD (Replaced Token Detection)MLM + Dynamic Masking
학습 신호"이 단어가 진짜인가 가짜인가""빈칸에 어떤 단어가 올까"
강점단어 수준 이상 탐지문장 간 관계/맥락 파악
NSP 태스크—제거 → 문맥 집중
KLUE NLI/STS약간 낮음상위
파라미터109M110M (동등)
선택 근거: 우리 과제의 핵심은 "같은 단어가 다른 맥락에서 쓰일 때 구분하는 것".
  • KcELECTRA의 RTD는 단어 수준 판별에 편향 → 문체 숏컷의 원인
  • RoBERTa의 MLM + Dynamic Masking은 주변 문맥에서 의미를 추론하도록 학습
  • "죽겠다" 주변에 "일", "피곤" 이 있으면 비유, "칼", "살려" 가 있으면 위협 — 이 구분은 문맥 파악 능력에 의존

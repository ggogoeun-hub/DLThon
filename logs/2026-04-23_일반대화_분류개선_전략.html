핵심 문제: Validation에서는 99%인데 Test에서 0.03. 이 격차는 모델이 "문체 숏컷"을 학습했다는 강력한 증거. 일반 대화의 내용이 아니라, 합성 데이터 특유의 스타일을 외웠다.

2. 원인 분석 — 합성 일반 대화와 실제 Test의 문체 괴리

수치 비교: 모델이 학습한 "일반 대화" vs 실제 Test

특성합성 일반 (학습)Test 전체위협 (학습)괴리
존댓말 0.8회/건 3.7회/건 4.0회/건 4.6배 부족
물음표 (?) 2.2개 3.4개 4.2개 1.5배 부족
마침표 (.) 0.4개 5.0개 6.2개 12.5배 부족
느낌표 (!) 0.0개 0.8개 1.1개 전무
길이 174자 212자 221자 22% 짧음
모델이 학습한 규칙 (의도하지 않은 숏컷):
  • "마침표가 많으면 → 위협" (합성 일반 0.4 vs 위협 6.2)
  • "존댓말이 있으면 → 위협" (합성 일반 0.8 vs 위협 4.0)
  • "느낌표가 있으면 → 위협" (합성 일반 0.0 vs 위협 1.1)
  • "길면 → 위협" (합성 일반 174자 vs 위협 221자)

→ Test의 일반 대화는 마침표·존댓말·느낌표가 있는 정상 문체 → 모델이 전부 위협으로 오분류

실제 텍스트 비교

합성 일반 대화 (학습 데이터)

"이 원피스 결혼식에 입어도 되나? 화이트인데 그건 안 돼 신부 색이잖아 하객은 화이트 피해야 해 아 맞다 그럼 뭘 입지"

→ 반말, 마침표 없음, 짧은 문장, 캐주얼

Test의 일반 대화 (실제)

"야 박보영 오늘 학원 갔다왔어 아니 오늘 못갔어 왜 아니 친구랑 놀다가 시간을 못봤어 자랑이냐 깜빡했어 저녁은 뭐야 너는 저녁 없어 학원 다녀왔어야지 엄마 미안해 나 배고파"

→ 구어체, 맥락 전환, 가족 대화, 자연스러운 톤

핵심: 합성 데이터는 "친구 간 정보 교환" 스타일만 집중. 실제 test에는 가족 대화, 일상 잡담, 감정 표현 등 다양한 유형이 포함되어 있음.

전략 B. 증강 비율 축소 권장

×3.7 증강이 오히려 증강 패턴 과적합을 유발했을 가능성.

옵션위협 증강일반 합성총 데이터비고
현재3,000 (×3.4)3,00015,000과적합 위험

1위 팀(0.79)도 제한된 데이터로 달성. 과도한 증강보다 적은 양의 고품질 데이터가 더 효과적일 수 있음.

전략 C. 빠른 실험 — 증강 없이 원본+합성만 즉시 실행 가능

가장 빠르게 test 결과를 확인할 수 있는 방법:

  1. 원본 train (3,846건, 증강 없음)
  2. + 합성 일반 대화 ~1,000건 (기존 합성 중 품질 좋은 것만)
  3. = ~4,800건으로 학습
  4. MAX_LEN=128로 속도 확보 → 1~2시간 내 결과

이전 실험(S02)에서 이 구성으로 Val F1 0.94 달성. 증강 과적합 제거 시 test 성능 개선 기대.

5. 핵심 교훈

Val F1 99%는 함정이었다.
모델은 "대화 내용"이 아니라 "합성 데이터의 문체적 특징"(마침표 없음, 반말 위주, 짧은 길이)을 학습했다. Validation은 같은 합성 데이터 분포이므로 높은 점수가 나오지만, 실제 Test에서는 문체가 다르므로 완전히 실패한다.
데이터 중심 AI에서의 교훈: 합성 데이터의 "내용"만이 아니라 "형식적 특성"(구두점, 말투, 길이)까지 실제 데이터 분포와 일치시켜야 한다. EDA에서 발견한 수치 기준을 합성 데이터에 적용하되, test 데이터의 특성도 반드시 교차 확인해야 한다.

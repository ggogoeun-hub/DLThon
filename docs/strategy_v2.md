## 2. 핵심 원칙 (v2)

### 일반 대화의 정의 (correct.md 기반)

```
일반 대화 = 쌍방 대등 + 유머/장난 톤 + 대화 마무리가 평화로움
```

**비속어가 있어도 일반 대화인 경우:**
- 친구 간 장난: "야 이 새끼 빨리 나와" → 대등, 유머
- 관용 표현: "야 죽을래" → 설거지 안 한 동생에게 (위협 의도 없음)
- 자조적 표현: "씨발 다들 잘갔네 부럽다" → 취업 스트레스 하소연

**일반 대화가 아닌 경우:**
- 일방적 비하: 한쪽이 반복적으로 공격, 상대가 굴복 (→ 괴롭힘)
- 위협 의도: "좋은말로 할 때 줘라" → 결국 물건 빼앗김 (→ 갈취)
- 권력 비대칭: 상사가 일방적으로 모욕, 부하가 "죄송합니다" 반복 (→ 직장 괴롭힘)

### 판별 기준 요약

| 기준 | 일반 대화 | 위협 대화 |
|---|---|---|
| 관계 | **쌍방 대등** | 가해자 → 피해자 |
| 거친 표현 | 장난/유머 톤 | 실제 위협 의도 |
| 마무리 | **평화롭게 종료** | 굴복/에스컬레이션 |
| 비속어 | 양쪽 다 사용 (대등) | 한쪽만 사용 (권력) |
| 감정 흐름 | 평탄 또는 해소 | 후반부 급증 |

## 3. 데이터 전처리

> 근거: strategy_v1.md 1절 + B01~B02 실험 결과

### 2-1. 중복 제거

| 작업 | 방법 | 상태 |
|---|---|---|
| 완전 중복 제거 | `df.drop_duplicates(subset=['conversation'])` | ✅ 구현 (104건 제거 → 3,846건) |
| 준-중복 제거 | TF-IDF cosine similarity ≥ 0.95인 쌍 제거 | ⚠️ 미구현 (117건 추정) |

### 2-2. 텍스트 정규화

> **핵심 전처리**: Train 100% `\n` 포함 vs Test 4.8%만 `\n` 포함 → 통일 필수

```python
def preprocess(text):
    text = str(text).replace('\n', ' ')       # newline → 공백
    text = re.sub(r'\s+', ' ', text).strip()  # 다중 공백 제거
    return text
```

| 항목 | 처리 | 근거 |
|---|---|---|
| `\n` 턴 구분자 | 공백으로 변환 | Train/Test 포맷 불일치 해소 (EDA 핵심 발견) |
| 다중 공백 | 단일 공백으로 | 정규화 |

> v1에서 `[SEP]` 토큰 방식 검토 후 공백 방식으로 확정. `[SEP]`는 BERT/ELECTRA의 세그먼트 구분 용도로 사전학습되어 대화 턴 구분과 의미 충돌 우려.

### 2-3. 특수문자/이모티콘 처리

> 근거: strategy_v1.md `[5-1]` — 원본 train의 emoticon_len = 0.0, ellipsis_count = 0.0

| 특수문자 | 처리 | 근거 |
|---|---|---|
| `ㅋㅋ`, `ㅎㅎ`, `ㅠㅠ` | **합성 데이터에서 사용 금지** | 원본 train에 없음. 포함 시 숏컷 유발 |
| `...` 말줄임 | **합성 데이터에서 사용 금지** | 원본 train에 없음 |
| `!` 느낌표 | 유지 | 클래스 구분 신호 (협박 1.85 > 갈취 1.02) |
| `?` 물음표 | 유지 | 전 클래스 4.08~4.87 고르게 분포 |

### 2-4. 불용어(조사) 처리

| 전략 | 설명 |
|---|---|

생성된 데이터는 여전히 4축 검증을 통과해야 함. 단, v2에서는 도메인 특성을 반영한 유연한 기준:

| 축 | 엄격 적용 도메인 | 유연 적용 도메인 |
|---|---|---|
| 발화 대칭 (<1.5) | 친구 수다, 커플 데이트, 동료 수다 | 부모-자녀, 119 신고, 감정 상담 |
| 존댓말 대칭 | 친구, 커플, 형제 | 부모-자녀(자연 비대칭), 알바-고객 |
| 에스컬레이션 없음 | 대부분 | 감정 상담 (격앙→위로→해소 허용) |
| 굴복 표현 없음 | 대부분 | 알바/서비스 ("죄송합니다" 정상 사용) |

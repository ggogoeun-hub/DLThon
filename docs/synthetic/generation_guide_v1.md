## 0. 왜 이 가이드가 필요한가

### 실험적 근거

| 실험 | 접근 | 결과 | 교훈 |
|---|---|---|---|
| B01 | "자연스러운 일상 대화" 생성 | Test 일반 34/100 | 프롬프트가 모호하면 도메인 갭 발생 |
| S01 | 길이만 개선 (139→206자) | 34→35 (변화 없음) | 표면 스펙만으로는 불충분 |
| S02 | 양 확대 (694→1,200건) | 34→38 (+4건) | 다양성 확보로 미미한 효과 |
| B02 | 15,000건 균형 + 이모티콘 제거 | Val 0.990 / Test **23건** | 문체 숏컷 학습 (느낌표/마침표/존댓말 차이) |

**결론**: 합성 데이터의 **양·길이**가 아닌 **형식적 동질성**(위협 대화와 구분 불가능한 형식)이 핵심.

## 1. 생성 원칙

```
❌ "자연스러운 일상 대화를 만들어주세요"          → 도메인 갭 발생
❌ "위협의 반대인 평화로운 대화를 만들어주세요"    → 표면 특성이 달라져 숏컷 유발
✅ "위협 대화와 동일한 형식, 무해한 내용의 대화"    → 도메인 갭 최소화
```

모델이 분류 신호로 사용할 수 있는 것은 **내용(의미)**뿐이어야 하며, **형식(구두점, 말투 비율, 발화 구조)**은 위협/일반 사이에 차이가 없어야 한다.

## 2. 3-레이어 생성 제약 조건

### 레이어 1: 표면 (구두점/형식)

> 근거: strategy_v1.md `[3-2]`, B02 숏컷 분석

| 파라미터 | 설정값 | 위협 데이터 수치 | 숏컷 위험 |
|---|---|---|---|
| 글자 수 | 180~250자 | median 185~220 | B02: 합성 174자 vs 위협 221자 → "짧으면 일반" |
| 단어 수 | 45~60개 | median 47~55 | 글자 수와 연동 |
| 발화 횟수 | **10턴 (±2)** | 전 클래스 median=10 | — |
| 느낌표 (!) | **1~2개** | 평균 0.70~1.85 | B02: 합성 0.04 vs 위협 1.15 → ×26 차이 |
| 물음표 (?) | **3~5개** | 평균 4.08~4.87 | B02: 합성 2.22 vs 위협 4.15 → ×2 차이 |
| 마침표 (.) | **4~8개** | 평균 ~6.18 | B02: 합성 0.38 vs 위협 6.18 → ×16 차이 |
| 이모티콘/초성 | **절대 금지** | emoticon_len = 0.0 | 합성에만 존재 → 즉시 숏컷 |
| 말줄임 (...) | **절대 금지** | ellipsis_count = 0.0 | 합성에만 존재 → 즉시 숏컷 |

**핵심**: 느낌표·마침표·물음표 개수를 위협 데이터 범위에 맞추는 것이 B02 실패의 직접적 해결책.

### 레이어 2: 화용 (말투/관계)

> 근거: strategy_v1.md `[3-3]` `[4-2]`, profile_threat.md 축 2·축 4

| 파라미터 | 설정값 | 위협 수치 | 설계 근거 |
|---|---|---|---|
| 존댓말 비중 | **2.0~3.0회/대화** | 평균 1.39~3.22 | strategy_v1 `[3-3]` 중간값 |
| 반말 비중 | **6.5~8.0회/대화** | 평균 6.62~7.94 | strategy_v1 `[3-3]` 범위 일치 |
| **화법 대칭** | 양쪽 동일 화법 | 한쪽만 존대 45~55% | 축 2: 권력 비대칭 제거 |
| 비속어 | **0 (절대 금지)** | — | 비속어 → 위협으로 오분류 |
| **리액션** | "맞아", "그래", "좋아", "인정", "대박" | 굴복: "죄송합니다" | 축 4: 대등 표현 위주 |
| **거절** | "난 별론데", "다음에 하자", "글쎄" | 굴복: "못합니다", "안됩니다" | strategy_v1 `[3-3]` 수평적 거절 |
| **발화 대칭** *(v2참고)* | 1:1 Tiki-taka, 오차 20% 이내 | — | strategy_v1 `[3-2]` |

**핵심**: "양쪽 동일 화법 + 대등 리액션 + 수평적 거절"이 위협과의 화용적 차별점. 단, 존댓말/반말 **총량**은 위협 범위 내로.

### 레이어 3: 구조 (대화 흐름)

> 근거: strategy_v1.md `[3-6]` `[6-2]`, profile_threat.md 축 1·축 3

| 파라미터 | 설정값 | 위협 수치 | 설계 근거 |
|---|---|---|---|
| **발화 길이 비율** | A/B = 1.0~1.3 | 1.5~2.0 (비대칭 53~60%) | 축 1: 가해자 길게/피해자 짧게 패턴 탈피 |
| **에스칼레이션** | Flat (전 구간 일정) | 후반 증가 59% (협박) | 축 3: 뒤로 갈수록 거세지는 패턴 방지 |
| **역할 구조** | 쌍방향 (서로 주제 제안) | 일방향 (가해자→피해자) | strategy_v1 `[3-6]` 역할 비대칭 제거 |
| **주제 전환** | 자연스러운 전환 1~2회 | 하나의 갈등에 집중 | 위협은 단일 갈등, 일상은 화제 전환 |
| **감정선** | 평탄 (Flat Sentiment) | 0.26→0.64 급증 | strategy_v1 `[3-6]` 핵심 차별점 |

**핵심**: 위협 대화는 "한 명이 주도하고 뒤로 갈수록 격해지는" 서사 구조. 일반 대화는 "두 명이 번갈아 이끌며 평탄한" 구조.

## 4. Hard Negative 전략

> 근거: strategy_v1.md `[3-5]`, `[3-1]` 클래스 간 코사인 유사도

혼동 쌍별 Hard Negative 설계:

| 혼동 쌍 (유사도) | Hard Negative 방향 | 사용 가능 키워드 | 사용 금지 키워드 |
|---|---|---|---|
| 협박↔기타 괴롭힘 (0.87) | 감정적이지만 무해한 대화 | 제발, 진짜, 무슨 | 죽여, 죽어, 패버릴 |
| 갈취↔기타 괴롭힘 (0.84) | 돈 관련 일상 (더치페이 등) | 돈이, 만원, 빌려 | 내놔, 맞을래, 시키는 대로 |
| 직장 괴롭힘↔기타 괴롭힘 (0.83) | 정상 직장 피드백/지시 | 죄송합니다, 부장님 | 닥쳐, 꺼져 |

**원칙**: 위협 키워드를 **무해한 문맥**에 배치하여 모델이 키워드가 아닌 **문맥**으로 판단하도록 유도.

## 5. 도메인별 프롬프트 템플릿

### 5-1. 직장 일상 (30%)

```
다음 조건을 모두 만족하는 한국어 직장 일상 대화를 생성하세요:

[형식]
- 2명의 직장 동료가 대화. 10턴 내외 (±2). 총 180~250자.
- 각 발화를 줄바꿈(\n)으로 구분.
- 느낌표 1~2개, 물음표 3~5개, 마침표 4~8개 자연스럽게 포함.
- ㅋㅋ/ㅎㅎ/ㅠㅠ 이모티콘, ... 말줄임 절대 사용 금지.

[관계 — 가장 중요]
- 양쪽 모두 반말 또는 양쪽 모두 존댓말 (한쪽만 존대 금지).
- "맞아", "그래", "좋아" 같은 대등한 동의 표현 사용.
- "죄송합니다", "잘못했습니다" 같은 굴복 표현 절대 사용 금지.

[구조 — 가장 중요]
- 화자 A와 B의 발화 길이가 비슷 (한쪽이 1.5배 이상 길지 않게).
- 대화 처음부터 끝까지 감정 톤이 평탄 (뒤로 갈수록 격해지지 않음).
- 서로 번갈아 주제를 제안하거나 의견을 나눔 (일방적 지시 금지).

[내용]
- 점심 메뉴, 회의 일정, 퇴근 계획, 주말 이야기, 프로젝트 근황 등.
- "회사", "부장님", "과장님" 등 직장 키워드를 자연스럽게 포함.
- 위협, 갈취, 괴롭힘 요소가 전혀 없는 대화.
```

### 5-2. 학교 일상 (20%)

```
[형식] 직장 일상과 동일.
[관계] 동일 (대등, 화법 대칭).
[구조] 동일 (발화 대칭, Flat Sentiment).
[내용]
- 수업, 과제, 시험, 동아리, 학식, 캠퍼스 생활 등.
- "선배", "후배", "교수님", "과제" 등 학교 키워드 포함.
- 선후배 관계여도 반말↔반말 또는 존대↔존대로 대칭 유지.
```

### 5-3. 일상/서비스 (20%)

```
[형식] 동일.
[관계] 동일.
[구조] 동일.
[내용]
- 카페 주문, 택배 문의, 식당 예약, 매장 방문 등.
- "고객님", "주문", "손님" 등 서비스 키워드 포함.
- 정상적인 서비스 상호작용 (클레임/갈등 아닌 것).
```

### 5-4. 친밀 관계 (15%)

```
[형식] 동일.
[관계] 동일 (특히 화법 대칭 중요 — 한쪽 지배 금지).
[구조] 동일.
[내용]
- 연인/친구 간 약속, 선물, 여행 계획, 근황 공유 등.
- 건강한 애정 표현 (집착/통제 아닌 것).
- 의견 다를 때 "난 별론데", "다음에 하자" 같은 대등한 거절.
```

### 5-5. 자유 주제 (15%)

```
[형식] 동일.
[관계] 동일.
[구조] 동일.
[내용]
- 취미, 뉴스, 날씨, 음식, 운동, 여행 등 자유 토픽.
- 특정 도메인에 치우치지 않는 일상 잡담.
```

## 7. 품질 검증

생성된 모든 합성 데이터는 `validation.md`의 자동 검증을 통과해야 함:

1. **개별 검증**: 길이, 턴수, 금지 단어, 이모티콘, 발화 대칭비, 존댓말 비대칭, 굴복 표현
2. **배치 검증**: 전체 분포가 위협 데이터 범위 내인지 통계 비교
3. **수동 샘플링**: 도메인별 10건씩 사람이 읽고 자연스러움 확인

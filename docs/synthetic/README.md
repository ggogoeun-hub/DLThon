# 합성 데이터 설계 — 4축 구조 분석 기반

> strategy_v1.md 3절의 상세 구현. "어떻게 만들지"에 집중.

## 배경

합성 일반 대화 데이터의 양과 길이를 늘려도 Test 일반 대화 탐지율이 개선되지 않는 문제 확인 (B01~S02 실험). 원인은 합성 데이터와 실제 대화의 **구조적 도메인 갭** — 표면(구두점)이 아닌 화용(권력 관계, 감정 흐름) 수준의 차이.

## 4축 분석 프레임워크

위협 대화의 구조적 특성을 4개 축으로 정량화하고, 그 "반대 설계"로 합성 데이터 가이드라인 도출.

| 축 | 측정 대상 | 위협 프로파일 | 일반 목표 |
|---|---|---|---|
| 1. 발화 대칭성 | 화자 A/B 발화 길이 비율 | 비대칭 53~60% | 비대칭 < 20% |
| 2. 존댓말 비대칭 | 화자 간 존댓말 사용 차이 | 한쪽만 존대 45~55% | 비대칭 < 10% |
| 3. 에스칼레이션 | 전반/후반 위협 단어 밀도 차이 | 후반 증가 59% (협박) | 에스컬 < 20% |
| 4. 굴복/대등 | 굴복("죄송합니다") vs 대등("맞아") 비율 | 협박·직장: 1.02 | < 0.1 |

# gem_prompt_v2.md 수정 사항 (strategy_v2 크로스 체크)

## 수정이 필요한 6개 항목

---

### 1. 🔴 길이 하한 (가장 중요)

**현재** (gem_prompt_v2 마스터 프롬프트):
```
5. 대화 길이는 50자~500자 사이로 다양하게 생성
```
**문제**: 실제 생성 결과 평균 116자. LLM이 하한(50자)에 치우침. test 평균 212자 대비 절반.

**수정**:
```
5. 대화 길이는 150자~400자 사이로 생성. 100자 미만은 절대 안 됨. 최소 8턴 이상.
```
→ 도메인별 프롬프트(P01~P33)의 길이 범위도 하한을 150자로 일괄 상향.

---

### 2. 🔴 validation.md 비속어 기준 충돌

**현재** (validation.md):
```
| 비속어 | 0회 | 즉시 폐기 |
```
**문제**: strategy_v2에서 비속어 "빈도 제어하에 허용"으로 변경했는데, validation.md는 v1 기준 그대로 → 생성한 데이터의 20~30%가 검증에서 폐기됨.

**수정** (validation.md):
```
| 비속어 | "새끼" 30건 중 4~5건, "씨발" 0~1건 이내 | 빈도 초과 시 폐기 |
```

---

### 3. 🔴 validation.md 길이/턴 기준 충돌

**현재** (validation.md):
```
| 글자 수 | 150~400자 | 재생성 |
| 발화 턴 수 | 8~12턴 | 재생성 |
```
**문제**: strategy_v2는 "50~500자, 4~20턴"으로 유연화. 신고 전화(짧음)나 상담(김)이 걸러짐.

**수정** (validation.md):
```
| 글자 수 | 100~500자 | 재생성 |
| 발화 턴 수 | 4~20턴 | 재생성 |
```

---

### 4. 🟡 생성 수량 990건 → 3,000건

**현재** (gem_prompt_v2):
```
총량: 33개 도메인 × 30개 = 990건
```
**변경 결정**: 증강 없이 직접 3,000건 생성하기로 함.

**수정**:
```
총량: 33개 도메인 × 91개 = 3,003건 (증강 없이 직접 생성)
```
→ 도메인별 프롬프트의 "30개 생성" → "91개 생성" (또는 30개씩 3회 요청)
→ strategy_v2 파이프라인 Step 4(증강) 삭제 또는 선택사항으로 변경

---

### 5. 🟡 금지 단어 목록 vs 관용 표현 허용 충돌

**현재** (validation.md BANNED_WORDS):
```python
BANNED_WORDS = ['죽여', '죽어', '맞을래', '시키는 대로', '내놔', '닥쳐', '패버릴']
```
**문제**: strategy_v2에서 "죽을래", "확 때릴라" 같은 관용 표현 허용. 그런데 "죽"이 부분 매칭으로 걸릴 수 있음.

**수정** — 금지 단어를 정확 매칭으로 변경하고, 관용 표현 예외 추가:
```python
BANNED_WORDS = ['죽여버린다', '죽여줄까', '칼로 찌른다', '시키는 대로', '내놔', '닥쳐', '패버릴']
# 허용: '죽을래'(장난), '죽겠다'(비유), '때릴라'(장난)
```

---

### 6. 🟢 마스터 프롬프트에 "최소 턴 수" 명시

**현재**: "최소 4턴, 최대 20턴"
**문제**: 4턴이면 2왕복뿐이라 대화가 너무 빈약. 실제 test 평균 ~10턴.

**수정**:
```
2. 2명의 멀티턴 대화로 작성 (최소 8턴, 최대 20턴)
```
→ P24(카톡 스타일)만 예외적으로 "최소 6턴" 허용

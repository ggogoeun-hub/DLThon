## 총평

EDA → 전략 수립 → 모델링의 흐름이 체계적이고, 특히 **strategy.md의 데이터 전략**은 EDA 수치 근거를 일일이 달아둬서 설득력이 높습니다. 대회 평가 기준(EDA 깊이, 논리적 근거, 실험 다양성)에 잘 맞춘 구조입니다.

아래는 개선하면 좋을 점들입니다.

## 2. 개선 필요 사항

### 2-1. 모델 관련

**`[SEP]` 토큰 충돌 문제** ⚠️ 중요
- 현재: 대화 턴 구분에 `[SEP]` 사용 (`\n` → `[SEP]`)
- 문제: BERT 계열에서 `[SEP]`는 **세그먼트 구분** 용도로 사전학습됨. 대화 턴 구분과 의미가 달라서 모델이 혼란할 수 있음
- 제안: `[USR1]`/`[USR2]` 같은 **special token을 추가 등록**하거나, 단순히 ` | ` 같은 구분자 사용 후 ablation으로 비교

**MAX_LEN=256 → 75% 커버리지**
- 25%의 대화가 잘림. 특히 긴 대화가 많은 협박 클래스(평균 246자, 최대 874자)에서 정보 손실 가능
- 제안: 512로 올린 실험을 ablation에 반드시 포함. 메모리 이슈 시 gradient accumulation으로 보완

**[CLS] 풀링만 사용**
- 대화 데이터는 여러 턴에 걸쳐 위협이 에스칼레이션되므로, 마지막 토큰이나 mean pooling이 더 나을 수 있음
- 특히 `[6-2]`에서 발견한 "후반부 위협 급증" 패턴을 [CLS]가 충분히 포착하는지 의문
- 제안: Mean pooling, Attention pooling과 ablation 비교

**5 에폭은 보수적**
- 3,800~5,000건 데이터에 5에폭이면 underfitting 가능성
- 제안: Early stopping (patience=3) + 최대 10~15에폭으로 설정

### 2-2. 데이터 전략 관련

**문장 순서 셔플 증강은 위험** ⚠️
- `[6-2]` 분석에서 "후반부로 갈수록 위협 밀도 증가"를 핵심 패턴으로 발견했는데, 문장 셔플은 이 패턴을 파괴함
- 자체 분석과 자체 전략이 모순됨
- 제안: 셔플 대신 **패러프레이즈(동일 의미 다른 표현)** 증강으로 대체

**합성 데이터 품질 검증 파이프라인 부재**
- 1,000건 합성 후 품질이 기준에 맞는지 자동 검증하는 코드가 없음
- 제안: 글자 수, 턴 수, 금지 단어, 비속어 0 등 체크리스트를 자동 검증하는 validator 함수 작성

**역번역 구체성 부족**
- 어떤 번역 API/모델을 쓸 건지, 품질 필터링 기준이 없음
- 한→영→한 과정에서 한국어 특유의 존댓말/반말 뉘앙스가 소실될 수 있음
- 제안: 역번역 전후 cosine similarity가 일정 임계값(예: 0.85) 이상인 것만 채택

### 2-3. 코드/구현 관련

**파일명 불일치**
- `data/` 폴더에는 `train.csv`가 있는데, `model.ipynb`에서는 `baseline.csv`를 로드
- 전처리/증강 후 저장하는 파일이 `baseline.csv`인 것 같은데, 이 파이프라인 코드가 없음

**증강/합성 코드 미구현**
- `strategy.md`에 상세 전략이 있지만, 실제 구현 코드(`augment.py`, `synthesize.py` 등)가 없음
- 전략 → 구현 간 갭이 큼

**의존성 관리 없음**
- `requirements.txt` 또는 `pyproject.toml`이 없어서 재현이 어려움
- KoNLPy, transformers, torch 등 버전 명시 필요

**Cross-validation 미적용**
- ~4,000건 데이터에 단일 80:20 split은 분산이 큼
- 제안: 5-fold stratified CV로 안정적인 성능 측정 (최소 3-fold)

### 2-4. Ablation Study

**템플릿만 있고 결과 없음**
- Exp-1~6 설계는 잘 되어 있으나 아직 실행 전
- 대회 평가에서 "실험 다양성"이 핵심 기준이므로 빠른 실행 필요

**추가 실험 제안**
- `Exp-7`: Focal Loss vs CrossEntropyLoss (클래스 혼동 완화)
- `Exp-8`: [CLS] vs Mean Pooling vs Attention Pooling
- `Exp-9`: [SEP] 턴 구분 vs custom special token
- `Exp-10`: KLUE-RoBERTa-base vs KLUE-RoBERTa-large vs KcELECTRA

## 4. 발표 전략 팁

- **"왜 이렇게 했는가"**가 핵심. strategy.md의 EDA 근거 기반 서사를 발표에 그대로 가져가면 강력함
- Hard Negative 전략은 차별화 포인트 — confusion matrix 개선 before/after를 시각화하면 효과적
- Ablation 결과 테이블은 반드시 포함 (실패한 실험도 "왜 실패했는지" 분석하면 가산점)

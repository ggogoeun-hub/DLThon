# 3일차 개별 실험 기록

## M1-A — 모델 비교: KLUE-RoBERTa-base 🥇

📅 2026-04-24 · 담당:

🔵 **가설**
동일 조건(Mean Pooling, 차등 LR, MAX_LEN=256)에서 RoBERTa의 MLM + Dynamic Masking이 ELECTRA의 RTD보다 "같은 단어, 다른 맥락" 구분에 유리할 것이다.

🟣 **실험**
- 모델: klue/roberta-base (110M params)
- 데이터: train_final.csv (15,000건, 5×3,000 균형)
- 설정: 백본 LR=4e-6, 헤드 LR=2e-5, epochs=3, batch=32, max_len=256
- 풀링: Mean Pooling + LayerNorm → FC(768→256) → GELU → Dropout(0.3) → FC(256→5)
- 환경: Google Colab T4 GPU

🟢 **결과**
- Val F1 Macro: `0.9299` / Accuracy: `0.9306`
- 일반 대화 Val F1: `0.9949`
- Test 일반대화: `74건` / 100건
- Test 분포: 직장 113 / 기타 106 / 협박 105 / 갈취 102 / 일반 74
- 학습시간: 35.1분 (3 epochs)

🟡 **인사이트**
✅ 가설 채택. 3종 중 Val F1, Test 일반대화 모두 1위. 특히 혼동 쌍(갈취↔기타 괴롭힘)에서 가장 큰 격차 발생. RoBERTa의 문맥 예측(MLM) 방식이 단어 수준 판별(RTD)보다 위협/일반 맥락 구분에 효과적.
→ 다음: RoBERTa 고정, 2단계 학습 전략 비교 진행

## M1-B — 모델 비교: KcELECTRA-base 🥉

📅 2026-04-24 · 담당:

🔵 **가설**
KcELECTRA는 한국어 댓글/구어체로 사전학습되어 위협 대화(구어체)에 더 적합할 수 있다.

🟣 **실험**
- 모델: beomi/KcELECTRA-base (109M params)
- 나머지 M1-A와 동일

🟢 **결과**
- Val F1 Macro: `0.8916` → M1-A 대비 -3.8%p
- 일반 대화 Val F1: `0.9899`
- Test 일반대화: `56건` → M1-A 대비 -18건
- Test 분포: 기타 121 / 직장 117 / 협박 106 / 갈취 100 / 일반 56
- 학습시간: 34.8분

🟡 **인사이트**
❌ 가설 기각. 구어체 특화라는 장점이 DKTC 과제에서는 오히려 단점. RTD 방식이 "죽겠다"같은 단어에 표면적으로 반응하여 맥락 구분 실패. 갈취 F1(0.846)과 기타 괴롭힘 F1(0.815)에서 RoBERTa 대비 큰 격차.

## M1-C — 모델 비교: KR-ELECTRA-discriminator 🥈

📅 2026-04-24 · 담당:

🔵 **가설**
KR-ELECTRA는 KcELECTRA보다 다양한 한국어 웹 코퍼스로 학습되어 더 범용적인 성능을 보일 것이다.

🟣 **실험**
- 모델: snunlp/KR-ELECTRA-discriminator (110M params)
- 나머지 M1-A와 동일

🟢 **결과**
- Val F1 Macro: `0.9121` → KcELECTRA(0.892) 대비 +2.1%p, RoBERTa(0.930) 대비 -1.8%p
- 일반 대화 Val F1: `1.0000` (완벽)
- Test 일반대화: `67건` → KcELECTRA(56) 대비 +11건, RoBERTa(74) 대비 -7건
- Test 분포: 기타 126 / 직장 108 / 협박 103 / 갈취 96 / 일반 67
- 학습시간: 35.0분

🟡 **인사이트**
⚠️ 부분적. KcELECTRA보다는 확실히 우위(+2.1%p)이지만 RoBERTa에는 미치지 못함. 일반대화 Val F1=1.000 완벽하지만 Test 67건 → 여전히 도메인 갭. ELECTRA 아키텍처 내에서는 사전학습 코퍼스 다양성이 중요함을 확인.

## M2-A — 단일 LR (차등 학습률 제거)

📅 2026-04-24 · 담당:

🔵 **가설**
차등 학습률(백본 4e-6 / 헤드 2e-5) 없이 전체 동일 LR(2e-5)로 학습하면, 백본의 사전학습 지식이 손상(Catastrophic Forgetting)되어 성능이 하락할 것이다.

🟣 **실험**
- 변경점: 백본 LR 4e-6 → 2e-5 (헤드와 동일, 차등 비율 1배)
- 나머지 M1-A와 동일

🟢 **결과**
- Val F1 Macro: `0.9299` → M2-B와 동일
- Test 일반대화: `69건` → M2-B(74건) 대비 -5건
- Test 분포: 직장 114 / 기타 109 / 갈취 106 / 협박 102 / 일반 69

🟡 **인사이트**
⚠️ Val은 동일하지만 Test 일반대화가 -5건 하락. 차등 학습률 없이 백본을 빡세게 학습시키면 "죽겠다=비유" 같은 맥락 구분 능력이 손상됨. 차등 학습률의 필요성 확인.

---

## M2-C — 지식보존형 LR (백본 2e-6, 차등 10배)

📅 2026-04-24 · 담당:

🔵 **가설**
백본 LR을 4e-6에서 2e-6으로 더 낮추면(차등 비율 5배→10배), 사전학습 지식이 더 잘 보존되어 맥락 구분 능력이 향상될 것이다.

🟣 **실험**
- 변경점: 백본 LR 4e-6 → 2e-6 (차등 비율 10배)
- 나머지 M1-A와 동일

🟢 **결과**
- Val F1 Macro: `0.9214` → M2-B(0.9299) 대비 -0.9%p
- 일반대화 Val F1: `0.9949`
- Test 일반대화: `77건` → **M2-B(74건) 대비 +3건, 2단계 LR 그룹 최고**
- Test 분포: 직장 111 / 협박 105 / 기타 104 / 갈취 103 / 일반 77

🟡 **인사이트**
✅ **핵심 발견: 차등 비율이 클수록 Test 일반대화 탐지율이 높아짐.**

| 차등 비율 | 백본 LR | Test 일반대화 |
|---|---|---|
| 1배 (M2-A) | 2e-5 | 69건 |
| 5배 (M2-B) | 4e-6 | 74건 |
| 10배 (M2-C) | 2e-6 | **77건** |

Val F1은 약간 낮아지지만, 실제 Test에서 맥락 구분 능력이 가장 우수. "백본의 한국어 맥락 파악 능력은 건드리지 않을수록 좋다"가 수치로 증명됨.

→ **백본 LR 2e-6 채택** (차등 비율 10배)

## M2-E — Label Smoothing 0.1 적용

📅 2026-04-24 · 담당:

🔵 **가설**
Label Smoothing(0.1)을 적용하면 모델이 예측에 과신하지 않아 일반화 성능이 향상될 것이다. B02에서 Val 0.990 → Test 0.03이었던 과적합 문제를 완화할 수 있다.

🟣 **실험**
- 변경점: CrossEntropyLoss(label_smoothing=0.1) 적용
- 나머지 M1-A와 동일 (백본 4e-6)

🟢 **결과**
- Val F1 Macro: `0.9341` → **2단계 전체 최고 (+0.4%p vs M2-B)**
- 일반대화 Val F1: `0.9975`
- Test 일반대화: `77건` → M2-B(74건) 대비 +3건
- Best Epoch: 2 (과적합 전에 일찍 수렴)
- Test 분포: 협박 116 / 직장 109 / 기타 106 / 갈취 92 / 일반 77

🟡 **인사이트**
✅ **효과 확인.** Val F1과 Test 일반대화 모두 상승. Label Smoothing이 모델의 과신을 방지하여 경계선 대화(일반↔위협)를 더 신중하게 판단하게 만듦. Best Epoch=2인 것도 "빠르게 수렴하고 과적합 안 됨"의 증거.

→ **Label Smoothing 0.1 채택**

---

## M2-G — [CLS] Pooling (Mean Pooling 제거)

📅 2026-04-24 · 담당:

🔵 **가설**
Mean Pooling은 대화 전체를 균등하게 반영하여 "마무리 부분의 중요 정보"가 희석될 수 있다. [CLS] 토큰이 전체 문맥을 더 잘 응축한다면 성능이 비슷하거나 더 높을 것이다.

🟣 **실험**
- 변경점: Mean Pooling → [CLS] Pooling (첫 번째 토큰의 hidden state 사용)
- 나머지 M1-A와 동일

🟢 **결과**
- Val F1 Macro: `0.9248` → M2-B(0.9299) 대비 -0.5%p
- 일반대화 Val F1: `0.9873` → Mean(0.9949) 대비 -0.8%p
- Test 일반대화: `73건` → M2-B(74건) 대비 -1건
- Test 분포: 직장 112 / 협박 109 / 기타 104 / 갈취 102 / 일반 73

🟡 **인사이트**
✅ **Mean Pooling이 우위 확인.** 대화 분류에서는 [CLS] 한 토큰보다 전체 토큰의 평균이 더 풍부한 정보를 제공. 특히 일반대화 Val F1에서 큰 차이(0.995 vs 0.987) — Mean이 대화 전체의 "톤"을 더 잘 잡아냄.

→ **Mean Pooling 유지**

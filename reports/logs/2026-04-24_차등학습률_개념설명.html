차등 학습률 (Discriminative Learning Rate)

DKTC 한국어 위협 대화 분류 실제 사례로 이해하기 | 2026-04-24

1. 비유: 경력자와 신입사원

모델 = 회사라고 생각하면
모델 전체 (110M 파라미터)
│
├── 백본 (RoBERTa)        ← "경력자" (이미 한국어를 잘 앎)
│   110,600,000개 (99.8%)
│   └── LR = 4e-6       → 살살 가르침 (기존 지식 망가뜨리면 안 되니까)
│
└── 분류 헤드 (FC층)      ← "신입사원" (위협/일반 구분은 처음)
    198,000개 (0.2%)
    └── LR = 2e-5       → 빡세게 가르침 (빨리 배워야 하니까)

LR 비율 = 2e-5 / 4e-6 = 5배 차이. 신입에게 5배 더 강하게 교육하는 셈.

2. 왜 차등 학습률이 필요한가

RoBERTa 백본은 이미 수십억 개의 한국어 문장으로 학습되어 있습니다.

"죽다", "돈", "회사" 같은 단어의 의미와 문맥을 이미 알고 있어요.

이 상태에서 학습률을 어떻게 주느냐에 따라 결과가 달라집니다:

백본 LR이 크면 (2e-5) — 단일 학습률

"죽겠다" → 원래 "힘들다는 비유"도 알고 있었는데
→ 학습 데이터에서 "죽" = 협박으로 많이 봄
→ 기존 지식이 덮어씌워짐 (Catastrophic Forgetting)
→ "야 배고파 죽겠다"도 협박으로 분류

백본 LR이 작으면 (4e-6) — 차등 학습률

→ 기존 한국어 지식은 보존하면서
→ "이 과제에서 중요한 부분만" 미세하게 조정
→ "죽겠다"의 다양한 맥락을 여전히 구분 가능
→ "배고파 죽겠다" = 일상, "죽여버린다" = 협박

핵심: 사전학습된 모델은 이미 "언어를 이해하는 능력"을 갖고 있습니다. 이 능력을 유지하면서 "우리 과제에 맞게만" 살짝 조정하는 것이 차등 학습률의 목적입니다.

3. DKTC에서의 실제 사례

같은 단어, 다른 맥락 — 이걸 구분해야 합니다

표현일반 대화 (무해)위협 대화 (유해)
"죽겠다" "배고파 죽겠다" → 비유 "죽여버리기 전에 말해" → 실제 위협
"돈" "만원만 빌려줘" → 친구 사이 "돈 안 내면 죽인다" → 갈취
"야 죽을래" 설거지 안 한 동생에게 → 관용 표현 "한번만 더 그러면 죽을래?" → 협박
"부장님" "부장님 점심 뭐 드실래요?" → 직장 일상 "부장님이 또 인신공격을..." → 직장 괴롭힘
RoBERTa 백본은 이미 이런 맥락 차이를 이해하는 능력을 갖고 있습니다. 차등 학습률은 이 능력을 망가뜨리지 않으면서 "5개 클래스로 분류하는 법"만 새로 가르치는 전략입니다.

4. 코드로 보면

단일 학습률 (모두 같은 속도로 학습)

# 전체 파라미터에 동일한 LR 적용
optimizer = AdamW(model.parameters(), lr=2e-5)
# → 백본도 2e-5, 헤드도 2e-5
# → "경력자도 신입처럼 빡세게" → 기존 지식 손상 위험

차등 학습률 (역할에 맞는 속도로 학습)

# 백본과 헤드를 분리
optimizer = AdamW([
    {'params': backbone_params, 'lr': 4e-6},  # 살살
    {'params': head_params,     'lr': 2e-5},  # 빡세게
])
# → 백본 지식 보존 + 헤드 빠른 학습

5. 2단계 Ablation Test에서 검증하는 것

"차등 학습률이 정말 효과가 있는지" 빼보고 비교합니다:

실험백본 LR헤드 LR비유질문
M2-A 2e-5 2e-5 경력자도 빡세게 "차등 없이도 괜찮아?"
M2-B 4e-6 2e-5 경력자 살살, 신입 빡세게 기준선 (B04 원본)
M2-C 2e-6 2e-5 경력자 더더 살살 "더 보수적이면 더 나아?"

예상 시나리오:

M2-A 성능 하락 → "차등 학습률이 필수다" 증명
M2-B ≈ M2-C → "4e-6이면 충분, 더 낮출 필요 없다"
M2-C > M2-B → "백본을 더 보수적으로 다뤄야 한다"

이것이 Ablation Study의 가치: "남들이 쓰니까"가 아니라 "빼봤더니 성능이 X%p 떨어졌으니까"라고 근거를 댈 수 있습니다.

6. 왜 이 개념이 중요한가

Transfer Learning의 핵심 딜레마

사전학습 모델을 사용하는 모든 NLP/CV 프로젝트에서 마주치는 질문:

"기존에 배운 것은 얼마나 유지하고, 새로운 것은 얼마나 빠르게 배울 것인가?"

유지를 너무 많이 하면 → 새 과제에 적응 못 함 (Underfitting)
새 학습을 너무 많이 하면 → 기존 지식 손실 (Catastrophic Forgetting)

 차등 학습률은 이 균형점을 찾는 가장 실용적인 방법입니다.

DKTC 프로젝트에서의 의미

B02에서 Val F1 0.99인데 Test 23건이었던 이유 중 하나가 바로 이것일 수 있습니다. 모델이 "합성 데이터의 패턴"을 너무 빠르게 외워버리면서 "한국어 문맥을 이해하는 능력"이 손상되었을 가능성. 차등 학습률은 이런 과적합을 구조적으로 방지합니다.

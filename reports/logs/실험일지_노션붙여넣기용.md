# 개별 실험 기록

## B01 — Baseline: KcELECTRA + 합성 일반대화 694건

📅 2026-04-22 · 담당:

🔵 **가설**
KcELECTRA-base는 한국어 댓글(구어체)로 사전학습되었으므로 위협 대화 분류에 적합할 것이다. Claude로 생성한 일반 대화 694건이면 5클래스 분류가 가능할 것이다.

🟣 **실험**
- 모델: beomi/KcELECTRA-base (109M params)
- 데이터: 원본 3,846건 + 합성 694건 (avg 139자) = 4,540건
- 설정: lr=2e-5, epochs=7(early stop), batch=16, max_len=128, CPU
- 전처리: \n → 공백 변환 (Train/Test 포맷 통일)

🟢 **결과**
- Val F1 Macro: `0.922` / Accuracy: `0.918` / 학습시간: 31분
- 일반 대화 Val F1: `0.995` (P=1.0, R=0.99)
- Test 예측: 협박 108 / 갈취 121 / 직장 119 / 기타 118 / ❌ 일반 34

🟡 **인사이트**
Val에서 일반 대화를 거의 완벽하게 분류하지만 Test에서는 34건만 예측. 합성 데이터와 실제 Test 일반 대화 사이에 도메인 갭 존재. 합성 데이터의 길이가 너무 짧아서(avg 139자 vs 원본 avg 227자) 길이 기반 shortcut 학습 가능성.
→ 다음: 합성 데이터 길이를 200자 이상으로 개선

## S01 — 합성 데이터 길이 개선 (avg 139자 → 206자)

📅 2026-04-22 · 담당:

🔵 **가설**
합성 일반 대화의 평균 길이가 원본과 유사해지면(139자 → 206자) 모델이 길이 shortcut 대신 내용 기반으로 학습하여 Test 일반 대화 탐지율이 올라갈 것이다.

🟣 **실험**
- 변경점: 합성 데이터를 v2로 재생성 (10~16개 발화, 평균 230자)
- 데이터: 원본 3,846건 + 합성v2 700건 (avg 206자) = 4,546건
- 나머지 B01과 동일

🟢 **결과**
- Val F1 Macro: `0.922` → B01과 동일
- Test 일반 대화 예측: `35건` → 34 → 35, 변화 없음

🟡 **인사이트**
❌ 가설 기각. 길이 개선만으로는 일반 대화 탐지율이 개선되지 않음. 문제는 길이가 아니라 합성 데이터의 스타일/다양성과 실제 Test 데이터의 도메인 차이.
→ 다음: 합성 데이터 양을 대폭 늘려서(694 → 1200) 다양성 확보 시도

## S02 — 합성 데이터 양 확대 (694건 → 1200건)

📅 2026-04-22~23 · 담당:

🔵 **가설**
합성 데이터 양을 거의 2배로 늘리면(694 → 1200) 더 다양한 일반 대화 패턴을 커버하여 Test 일반 대화 recall이 개선될 것이다.

🟣 **실험**
- 변경점: v2 batch1(500건) + hard negatives(200건) + v1 batch2(500건) = 1200건 결합
- 데이터: 원본 3,846건 + 합성 1,200건 (avg 187자) = 5,046건
- 추가: 디스크 부족 해결 (체크포인트 17GB 정리), save_total_limit=2 적용

🟢 **결과**
- Val F1 Macro: `0.921` → B01과 거의 동일
- 일반 대화 Val: P=1.0, R=1.0, F1=1.000 (완벽)
- Test 일반 대화 예측: `38건` → 34 → 38, +4건

🟡 **인사이트**
⚠️ 부분적 효과. 미미한 개선(+4건). 양을 늘리는 것만으로는 한계. Val에서 완벽(F1=1.0)한데 Test에서 38/100인 것은 전형적인 domain gap 문제. 합성 데이터의 "양"이나 "길이"가 아닌 "실제 대화와의 유사도"가 핵심 병목.
→ 다음 후보: (A) 모델 변경 (B) 임계값 조정 (C) 합성 데이터 프롬프트 전략 변경 (D) 데이터 증강

## S02b — MAX_LEN 확대 (128 → 256)

📅 2026-04-22 · 담당:

🔵 **가설**
MAX_LEN 128에서 잘리는 긴 대화의 정보가 손실되고 있다. 256으로 늘리면 대화 후반부 정보(에스칼레이션 패턴 포함)를 활용할 수 있어 F1이 개선될 것이다.

🟣 **실험**
- 변경점: max_len 128 → 256
- 데이터: S02와 동일 (원본 3,846건 + 합성 1,200건 = 5,046건)
- 나머지 S02와 동일

🟢 **결과**
- Val F1 Macro: `0.937` → S02 대비 +1.6%p
- 특이사항: 토큰 길이 128 커버율 ~95% → 256 커버율 99.4%

🟡 **인사이트**
✅ 가설 채택. MAX_LEN 확대만으로 +1.6%p. 다른 변수 변경 없이 순수하게 긴 대화 정보 활용 효과 확인. 이후 실험은 MAX_LEN=256을 기본값으로 고정.
→ 다음: 데이터 증강으로 클래스 균형 맞추기

## B02 — 대규모 증강 baseline.csv v2 (15,000건)

📅 2026-04-22~23 · 담당:

🔵 **가설**
위협 4클래스를 ×3.4배 증강하고 합성 일반 대화를 3,000건으로 확대하여 5클래스 × 3,000건 균형 데이터를 만들면, 클래스 불균형 해소 + 데이터 다양성 확보로 F1이 크게 개선될 것이다.

🟣 **실험**
- 변경점: 전면 데이터 재구축 (baseline.csv v2)
  - 원본 3,846건 → 위협 4클래스 각 3,000건 (경량 EDA 증강: 삭제+스왑+구두점)
  - 합성 일반 3,000건 (합성 CSV 5종 병합 + 이모티콘 제거 + 부족분 증강)
  - 총 15,000건 (5 × 3,000)
- 품질 개선: 이모티콘 전면 제거 (42.6% → 0%), 비문 증강 방식 교체
- 구조 검증: Train \n vs Test 공백 포맷 차이 확인 → 전처리 통일
- 설정: lr=2e-5, epochs=7(early stop patience=3), batch=16, max_len=256, Dynamic Padding, CPU
- 학습 시간: 5시간 5분

🟢 **결과**
- Val F1 Macro: `0.990` → S02b 대비 +5.3%p
- Val Accuracy: `0.990`
- 일반 대화 Val: P=1.0, R=1.0, F1=1.000 (완벽)
- Test 일반 대화 예측: `23건` → 이전 38건에서 오히려 감소 🔴
- Kaggle Test F1: `0.03` 🔴

**Confusion Matrix (Val):**
```
          갈취  기타  일반  직장  협박
갈취      442    3    0    0    5
기타        4  443    0    0    3
일반        0    0  450    0    0
직장        2    2    0  446    0
협박        2    1    0    0  447
```

🟡 **인사이트**
❌ 가설 기각 (Val 기준으로는 채택이지만, Test 기준으로는 완전 실패).

Val F1 99%는 **함정**이었다. 원인 분석 결과, 모델이 대화 "내용"이 아니라 합성 데이터의 "문체적 특징"을 숏컷으로 학습한 것을 확인:

| 숏컷 특성 | 합성 일반 | 위협 평균 | 배율 | 모델이 배운 규칙 |
|-----------|----------|----------|------|----------------|
| 느낌표 (!) | 0.04개 | 1.15개 | ×26 | "느낌표 없으면 일반" |
| 마침표 (.) | 0.38개 | 6.18개 | ×16 | "마침표 없으면 일반" |
| 존댓말 | 0.83회 | 4.05회 | ×5 | "반말 위주면 일반" |
| 물음표 (?) | 2.22개 | 4.15개 | ×2 | "물음표 적으면 일반" |
| 길이 | 174자 | 221자 | ×1.3 | "짧으면 일반" |

Test 일반 대화는 마침표·존댓말·느낌표가 정상적으로 포함되어 있어 → 모델이 전부 위협으로 오분류.

**추가 발견**: 증강 ×3.4배가 오히려 증강 패턴 과적합을 유발. 이전 S02(38건)보다 오히려 일반 대화 예측이 감소(23건).

→ 다음: (1) 증강 없이 원본+합성만으로 학습 (2) 합성 일반 대화 문체를 test 분포에 맞게 재생성

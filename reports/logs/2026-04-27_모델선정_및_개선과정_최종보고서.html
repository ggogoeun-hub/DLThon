2. Stage 1: 백본 모델 선정

2-1. 왜 사전학습 언어모델인가

Concept: Transfer Learning
대규모 한국어 코퍼스로 사전학습된 모델은 이미 한국어의 문법, 의미, 문맥을 이해합니다. 이 지식을 위협 대화 분류라는 특정 과제에 전이(transfer)하면, 3,950건의 적은 데이터로도 높은 성능을 달성할 수 있습니다.

대화 텍스트는 평균 116토큰(Test 기준)이므로, MAX_LEN=128이면 일부 대화가 잘리지만 256이면 대부분 수용 가능합니다.

확정: klue/roberta-base + MAX_LEN=256

3. Stage 2: 학습 전략 최적화

3-1. 차등 학습률 (Differential Learning Rate)

Concept: 왜 백본과 헤드의 학습률을 다르게 하는가
사전학습된 백본은 이미 한국어를 잘 이해하고 있으므로 조심스럽게 미세 조정(작은 LR)하고, 새로 추가한 분류 헤드는 처음부터 학습해야 하므로 더 큰 LR을 적용합니다. 이렇게 하면 백본의 사전학습 지식을 보존하면서 과제 특화 학습이 가능합니다. 
optimizer = AdamW([
    {'params': backbone_params, 'lr': 2e-6},   # 백본: 조심스럽게
    {'params': head_params,     'lr': 2e-5},   # 헤드: 적극적으로
])
# 차등 비율 = 2e-5 / 2e-6 = 10배

차등 비율에 따른 Test 일반대화 정분류 수:

차등 비율백본 LR헤드 LRTest 일반대화
1배 (동일)2e-52e-569건
5배4e-62e-574건
10배2e-62e-577건

차등 비율이 클수록 백본의 사전학습 표현을 보존하여 맥락 파악 능력 향상

3-2. 레이어 동결 (Layer Freezing)

Concept: RoBERTa의 12층 구조와 동결
RoBERTa는 12개의 Transformer 인코더 층으로 구성됩니다. 하위 층은 일반적인 언어 패턴(형태소, 구문)을, 상위 층은 과제 특화된 의미를 학습합니다. 하위 9층을 동결하면 일반 언어 지식을 보존하면서 상위 3층만 위협 대화 분류에 맞게 조정합니다. 
전체 파라미터: 111,114,248개 (111M)
동결 (하위 9층 + 임베딩): 88,763,904개 (80%)
학습 대상 (상위 3층 + 헤드): 22,350,344개 (20%)

3-3. 기타 하이퍼파라미터

항목탐색 범위최적값근거
Dropout0.1 / 0.3 / 0.50.30.1 ≈ 0.3 ≈ 0.5 차이 없음
Label Smoothing0 / 0.10.1Val +0.4%p, Test +3건

Concept: Label Smoothing
정답 라벨을 [0, 0, 0, 0, 1] 대신 [0.02, 0.02, 0.02, 0.02, 0.92]로 부드럽게 만듦. 모델이 하나의 클래스에 과도하게 확신하는 것을 방지하여 일반화 성능을 개선합니다.

확정: 차등 LR 10배 (Backbone 2e-6 / Head 2e-5) + LS 0.1 + Dropout 0.3 + 동결 9층

4. Stage 3: 인코더 출력 활용 전략 — 풀링 구조 실험

4-1. 문제 정의: 인코더 출력을 어떻게 요약할 것인가

Concept: 풀링 (Pooling)
RoBERTa는 입력 텍스트의 각 토큰에 대해 768차원 벡터를 출력합니다. "야 빨리 타라 늦으면 다 죽어"가 입력이면, 11개 토큰 × 768차원 = (11, 768) 행렬이 출력. 이 행렬을 하나의 벡터(768차원)로 요약하는 것이 풀링이고, 이 요약 방식이 분류 성능에 영향을 줍니다.

4-2. 3가지 풀링 전략

A. Ending Pooling — 마무리 톤 중점 분석

대화 토큰: [CLS] 야 빨리 타라 늦으면 다 죽어 내가 운전할 ... [SEP]
                                            └──── 마지막 64토큰 ────┘
                                                    ↓ 평균
                                              ending_pool (768d)

설계 의도: 대화의 결말이 "화해/수긍"이면 일반, "위협/굴복"이면 위협

B. Speaker-aware Pooling — 화자 분리 시도

토큰 포지션:  0    1    2    3    4    5    6    7
화자 배정:    A    B    A    B    A    B    A    B
              ↓              ↓
         pool(짝수)     pool(홀수)
              ↓              ↓
         concat [A_pool, B_pool] → 1536d

설계 의도: 화자 A/B를 분리하여 권력 비대칭 감지

C. Multi-head — 다관점 분석

mean_pool (768d) → equality_head → 스칼라 (대등성)
                  → ending_head   → 스칼라 (마무리)
                  → threat_head   → 스칼라 (위협)
                        ↓
concat [mean_pool(768), eq(1), end(1), threat(1)] = 771d → 분류

설계 의도: "대등성", "마무리 톤", "위협 의도"를 별도 헤드로 분석

5-2. Self-Attention이 하는 일

Concept: Self-Attention과 문맥 파악
같은 "죽어"라는 토큰이라도 주변 토큰에 따라 다른 벡터로 변환됩니다:

"늦으면 다 죽어 내가 운전할 테니까" → 게임 맥락의 "죽어" 벡터
"시키는 대로 안 하면 죽어" → 위협 맥락의 "죽어" 벡터

Self-Attention이 주변 토큰("게임", "운전" vs "시키는 대로")과의 관계를 계산하여, 동일 단어에 다른 의미를 부여합니다.

7-2. 청크 흐름 구조

가설: 대화의 전반→후반 변화(에스컬레이션 vs 해소)가 분류 핵심

first_half  = mean_pool(hidden[1 : L//2])     // 전반부
second_half = mean_pool(hidden[L//2 : L-1])   // 후반부
diff = second_half - first_half                // 변화 벡터
→ concat [first, second, diff] = 2304d → 분류

근거 — 같은 키워드, 다른 흐름:

일반: "새끼야 살빼" → "팩트라 할말없네"     = 수긍 (diff → 완화 방향)
위협: "새끼야 살빼" → "죄송합니다 그만해주세요" = 굴복 (diff → 비대칭 방향)
구두점제거 + Mean Pool구두점제거 + 청크 흐름
Val F10.88050.8965
Test 일반75건75건

결과: Val F1은 +1.6%p 개선되어 구조의 우위는 확인. 그러나 Test 일반대화는 동일 — 구두점 제거로 인한 정보 손실이 구조 개선을 상쇄.

7-3. 지배/복종 피처

가설: 위협 대화에는 "지배 표현 + 복종 표현"이 공존, 일반 대화에는 없음

클래스지배+복종 공존둘 다 없음
협박33.0%18.0%
갈취14.9%37.6%

매우 뚜렷한 신호. 그러나 5개 수동 피처 추가 시 Test 84→83건으로 유의미한 개선 없음 — 어휘 리스트 기반 피처의 커버리지 한계.

9-3. 핵심 교훈

1. 데이터 분포가 모델 구조를 압도한다

합성 일반대화의 길이/구두점 분포 불일치 하나가, 모델 구조 변경(3가지), 에폭 증가, 동결 해제보다 더 큰 영향. 구두점 제거, 청크 흐름, 피처 추가 등 모든 구조적 시도가 88건의 벽을 넘지 못함.

2. "Val 맹신 금지" — 검증 세트의 한계

Final Run: Val 0.9248(최고) → Test 77건(최저). Val 세트가 합성 데이터와 같은 분포이므로 Val 성능이 Test 일반화를 보장하지 않음. 동결 9층 모델이 과적합을 방지하여 오히려 Test에서 우수.

3. 설계 의도와 실제 구현을 코드 레벨에서 검증해야 한다

Speaker-aware(토큰 짝홀수 ≠ 화자 분리), Multi-head(같은 입력에서 스칼라 3개 ≠ 다관점 분석)가 설계 의도와 불일치. 구조를 만들었으면 실제로 의도한 정보를 보고 있는지 검증이 필수.

4. 숏컷은 성능 지표만으로 보이지 않는다

Val F1 0.91, Test 88건이면 "잘 작동한다"고 생각하기 쉽지만, 마침표별/길이별 분류율을 분석하면 숏컷 의존이 드러남. 예측 결과의 분포를 분석하는 습관이 중요.

5. 대화 분류의 본질은 "흐름"과 "역학"

개별 키워드("죽여", "새끼")가 아니라 청크 간 관계(수긍 vs 굴복, 에스컬레이션 vs 해소)가 위협과 일반을 가름. 같은 "새끼야 살빼"도 후반부가 "팩트라 할말없네"(대등)인지 "죄송합니다"(비대칭)인지에 따라 클래스가 달라짐.

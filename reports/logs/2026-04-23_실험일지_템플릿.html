개별 실험 기록

B01
Baseline — KcELECTRA + 합성 일반대화 694건
2026-04-22 · 담당: [이름]
가설

KcELECTRA-base는 한국어 댓글(구어체)로 사전학습되었으므로 위협 대화 분류에 적합할 것이다.

Claude로 생성한 일반 대화 694건이면 5클래스 분류가 가능할 것이다.

실험

모델: beomi/KcELECTRA-base (109M params)

데이터: 원본 3,846건 + 합성 694건 (avg 139자) = 4,540건

설정: lr=2e-5, epochs=7(early stop), batch=16, max_len=128, CPU

전처리: \n → 공백 변환 (Train/Test 포맷 통일)

결과

Val F1 Macro: 0.922   Accuracy: 0.918   학습시간: 31분

일반 대화 Val F1: 0.995 (P=1.0, R=0.99)

Test 예측: 협박 108 / 갈취 121 / 직장 119 / 기타 118 / 일반 34

인사이트

Val에서 일반 대화를 거의 완벽하게 분류하지만 Test에서는 34건만 예측.

→ 합성 데이터와 실제 Test 일반 대화 사이에 도메인 갭 존재. 합성 데이터의 길이가 너무 짧아서(avg 139자 vs 원본 avg 227자) 길이 기반 shortcut을 학습했을 가능성.

→ 다음 실험: 합성 데이터 길이를 200자 이상으로 개선하여 재학습.

S01
합성 데이터 길이 개선 (avg 139자 → 206자)
2026-04-22 · 담당: [이름]
가설

합성 일반 대화의 평균 길이가 원본과 유사해지면(139자 → 206자) 모델이 길이 shortcut 대신 내용 기반으로 학습하여 Test 일반 대화 탐지율이 올라갈 것이다.

실험

변경점: 합성 데이터를 v2로 재생성 (10~16개 발화, 평균 230자)

데이터: 원본 3,846건 + 합성v2 700건 (avg 206자) = 4,546건

나머지 설정: B01과 동일

결과

Val F1 Macro: 0.922 B01과 동일

Test 일반 대화 예측: 35건 34 → 35 (변화 없음)

인사이트

가설 기각. 길이 개선만으로는 일반 대화 탐지율이 개선되지 않음.

→ 문제는 길이가 아니라 합성 데이터의 스타일/다양성과 실제 Test 데이터의 도메인 차이.

→ 다음 실험: 합성 데이터 양을 대폭 늘려서(694 → 1,200) 다양성 확보 시도.

S02
합성 데이터 양 확대 (694건 → 1,200건)
2026-04-22~23 · 담당: [이름]
가설

합성 데이터 양을 거의 2배로 늘리면(694 → 1,200) 더 다양한 일반 대화 패턴을 커버하여 Test 일반 대화 recall이 개선될 것이다.

실험

변경점: v2 batch1(500건) + hard negatives(200건) + v1 batch2(500건) = 1,200건 결합

데이터: 원본 3,846건 + 합성 1,200건 (avg 187자) = 5,046건

추가 조치: 디스크 부족 해결 (체크포인트 17GB 정리), save_total_limit=2 적용

결과

Val F1 Macro: 0.921 B01과 거의 동일

일반 대화 Val: P=1.0, R=1.0, F1=1.000 (완벽)

Test 일반 대화 예측: 38건 34 → 38 (+4건)

인사이트

부분적 효과. 미미한 개선(+4건). 양을 늘리는 것만으로는 한계.

→ Val에서 완벽(F1=1.0)한데 Test에서 38/100인 것은 전형적인 domain gap 문제.

→ 합성 데이터의 "양"이나 "길이"가 아닌 "실제 대화와의 유사도"가 핵심 병목.

→ 다음 실험 후보: (A) 모델 변경 (B) 임계값 조정 (C) 합성 데이터 프롬프트 전략 변경 (D) 데이터 증강

실험 ID 규칙

B##Baseline / 모델 비교 (KcELECTRA, KcBERT, KLUE-RoBERTa 등)
S##합성 데이터 변경 (양, 길이, 프롬프트 전략, hard negative 등)
A##데이터 증강 (back-translation, 문장 셔플, synonym replacement 등)
H##하이퍼파라미터 튜닝 (lr, epochs, max_len, batch_size, warmup 등)
E##앙상블 (soft voting, hard voting, stacking 등)

주요 비교 지표

  • Val F1 Macro — 주 평가 지표 (Kaggle 평가 기준)
  • Test 일반 대화 예측 수 — 현재 최대 병목. 100건에 가까울수록 좋음
  • Test 예측 분포 균등성 — 5클래스 각 100건이 정답이므로 분포가 균등할수록 유리
  • 클래스별 Confusion — 어떤 클래스 쌍이 혼동되는지 추적

"Train에 없는 클래스(일반 대화)를 Test에서 100건 정확히 분류해야 한다"

솔루션 개요: 3-Layer 엔지니어링

Layer 1: 데이터  — "없는 데이터를 어떻게 만들 것인가"
Layer 2: 모델   — "만든 데이터로 어떻게 잘 학습시킬 것인가"
Layer 3: 검증   — "잘 되고 있는지 어떻게 믿을 것인가"

3일간의 시행착오(B01~M2-G, 15회 실험)를 통해 각 Layer의 핵심 원리를 수치로 검증했습니다.

Top-Down vs Bottom-Up

Top-Down (B01~B02): 실패

위협 데이터 분석 → "위협의 반대" 설계
→ "비속어 금지, 느낌표 제거, 평탄한 대화"
→ 실제 일반 대화와 동떨어진 합성 데이터
→ 모델이 "형식"으로 분류 (숏컷)

Bottom-Up (B04): 성공

"한국인의 일상 대화는 뭐가 있나?" → 33개 도메인
→ 비속어 포함, 119 신고, 감정 상담 등 다양
→ 위협 대화와 형식이 비슷하되 내용만 무해
→ 모델이 "내용"으로 분류 (맥락 파악)

윤리적 고려: Test 데이터 독립성

초기 접근(correct.md)에서는 Test 데이터를 분석하여 합성 전략을 수립했으나, 검증 독립성 우려로 방향을 수정했습니다.

최종 버전(gem_prompt_v2)의 33개 도메인은 Test 데이터를 참고하지 않고, Train 데이터 분석 + 한국어 일상 대화의 일반적 도메인 지식만으로 설계했습니다.

교훈: 3가지 검증 원칙

1. Val 맹신 금지
→ Test 일반대화 예측 수로 판단

2. Leak-proof 검증셋
→ 합성 프롬프트 6개를 물리적 격리

3. 숏컷 의심
→ 4축 구조 분석으로 형식 차이 감시

핵심 메시지

"없는 데이터를 만드는 것은 시작일 뿐이고,
진짜 도전은 모델이 데이터의 '내용'을 배우게 하되
'형식'을 외우지 않게 하는 것이었다"

발표 키 메시지 (3개)

1. 데이터: "위협의 반대"가 아닌 "실제 대화 유형의 체계적 재현"으로 도메인 갭 극복 (34건→84건)

2. 모델: 사전학습 지식은 "건드리지 않을수록 좋다" — 차등 비율 10배에서 최고 성능 (69건→77건)

3. 검증: Val F1 99%가 함정일 수 있다 — Test 기준 + 숏컷 감시 + 4축 구조 분석으로 진짜 성능 판단

예상 질의 대비

예상 질문답변 핵심
"Test 데이터를 보고 합성 전략을 세운 건 아닌가?" 최종 gem_prompt_v2는 Test 없이 Train 분석 + 한국어 상식만으로 설계. 비속어 허용, 33개 도메인 등은 한국어 화자라면 도출 가능한 일반적 지식.
"왜 RoBERTa가 ELECTRA보다 나은가?" MLM은 문맥 전체를 보고 예측, RTD는 개별 단어의 진위를 판별. "죽겠다"의 맥락 구분에는 문맥 예측 방식이 효과적. 수치: +3.8%p(Val), +18건(Test)
"차등 학습률이 왜 중요한가?" 1.1억 파라미터 중 99.8%인 백본이 이미 "한국어 맥락 파악"을 알고 있음. 이걸 보존하면서 0.2%의 분류층만 학습시키는 것이 핵심. 차등 비율 10배에서 Test 일반대화 최고(77건).
"Val 성능이 높으면 좋은 거 아닌가?" B02에서 Val 0.990이었지만 Test 23건. 합성 데이터끼리만 잘 구분하고, 실제 데이터는 못 구분. Leak-proof 검증셋 + Test 예측 분포로 이중 검증 필수.
